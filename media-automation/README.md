# 미디어 자동화 (media-automation)

영상 편집 · 썸네일 · 블로그 자동화 도구 모음.
지금은 **① 썸네일 자동 제작**이 먼저 완성돼 있습니다. (영상 편집 / 블로그는 다음 단계)

쉽게 말하면 — **사진 한 장과 "고정 틀"만 있으면, 매일 다른 문구로 유튜브·인스타 썸네일을 자동으로 찍어주는 도장 기계**예요.

---

## 1. 썸네일 자동 제작

### 무엇이 "고정"되고 무엇이 "매일 바뀌나"

| 고정되는 것 (틀) | 매일 바뀌는 것 (내용) |
|---|---|
| 글씨체·색·크기·배치 | 큰 제목 문구 |
| 배지(브랜드 이름) 위치 | 윗줄(후킹 문구) |
| 어두운 그라데이션 | 아랫줄(보조 설명) |
| 유튜브/인스타 각각의 비율 | 주제(신차/출고/시승…)·표현 방식 |

> 레퍼런스로 주신 인스타 썸네일 4장의 공통 스타일(큰 흰색 2줄 제목 + 윗줄 후킹 + 좌상단 배지 + 사진 위 어두운 처리)을 그대로 틀로 박제했습니다.

### 유튜브 ≠ 인스타 (서로 다르게 나옵니다)

- **인스타**: 세로 9:16 (1080×1920), 글씨가 **아래쪽**에 모임
- **유튜브**: 가로 16:9 (1280×720), 글씨가 **왼쪽**에 모이고 오른쪽엔 사진이 보임

### 쓰는 법 (2단계)

```bash
cd media-automation

# 1) 오늘의 문구 만들기 (중복 없이 자동 선택)
npm run pick

# 2) 사진 1장 → 유튜브·인스타 썸네일 2장 자동 생성
npm run thumb -- --bg <사진경로>
# 결과물: output/2026-06-30_youtube_*.png , output/2026-06-30_instagram_*.png
```

영상 주제에 맞춰 고정하고 싶으면:

```bash
npm run pick -- --topic newcar     # 신차 / delivery 출고 / testdrive 시승 ...
```

미리보기만(기록 안 남김):

```bash
npm run pick -- --date 2026-07-01 --dry
```

## 2. "매일 바뀌고 안 겹치게" 하는 기준

`config/content-rules.json` 의 `rules` 에서 관리합니다.

| 기준 | 기본값 | 뜻 |
|---|---|---|
| `noRepeatHeadlineDays` | 120 | 같은 제목은 120일 안에 다시 안 나옴 |
| `noSameTopicConsecutiveDays` | 1 | 어제와 같은 주제 금지 |
| `avoidLastNPatterns` | 2 | 최근 2번 쓴 "후킹 공식" 반복 금지 |
| `maxCharsPerHeadlineLine` | 13 | 제목 한 줄이 너무 길지 않게 |
| `keywordOverlapReject` | 0.6 | 최근 제목과 단어가 60% 넘게 겹치면 버림 |

이미 쓴 문구는 `state/used-content.json`(원장)에 자동 기록되어, 다음에 같은 게 안 나오도록 막습니다.

### 내용·주제를 바꾸고 싶을 때

`config/content-rules.json` 한 파일만 고치면 됩니다:

- `brand` — 배지에 뜨는 이름/핸들
- `topics` — 다루는 주제 목록
- `slots` — 차종·숫자·문구 재료 (여기에 단어를 더 넣을수록 더 다양해짐)
- `patterns` — 후킹 공식(숫자형·비교형·시간형·부정형·후기형·질문형)

색·크기·배치는 `config/thumbnail-style.json` 에서 조정합니다.

---

## 폴더 구조

```
media-automation/
  config/
    thumbnail-style.json   # 고정 틀(색·크기·배치)
    content-rules.json     # 문구 생성·중복방지 기준  ← 주로 여기만 수정
  templates/thumb.html     # 썸네일 디자인(HTML)
  scripts/
    pick-content.mjs       # 오늘의 문구 생성 + 중복 차단
    make-thumbnail.mjs     # 사진+문구 → PNG (유튜브/인스타)
  state/used-content.json  # 이미 쓴 문구 기록(자동)
  assets/                  # 폰트·샘플
  output/                  # 생성된 썸네일
```

## 준비물 (이 도구가 돌아가려면)

- Node.js 20+
- Chromium (썸네일을 정확한 크기로 찍기 위함)
- 한글 폰트: Noto Sans CJK KR (없으면 나눔스퀘어로 대체)
- ffmpeg (영상 편집 단계에서 사용 예정)

## 3. 영상 편집 자동화 (고정 폼)

원본 세로 영상 → **상단 인사 자막 + 하단 명함**을 입힌 편집본(인스타/유튜브)으로 자동 변환.
편집본 예시(`비오는날.mp4`, `0627(3).mp4`)에서 형태를 그대로 규격화했습니다.

### 고정 폼 + 4가지 스타일 (매일 로테이션)

- **세로 9:16** (1080×1920, 릴스/쇼츠)
- **자막 스타일 4종**(매일 자동으로 돌아가며 "다른 느낌"):
  - `bar` 상단 풀바 · `box` 상단 둥근박스 · `lower` 하단 좌측(노란 강조) · `side` 좌측 세로바
- **하단 명함**: 실제 명함 이미지 노출 (단, 하단 자막 스타일일 땐 겹침 방지로 생략)

### 쓰는 법

```bash
# 원본 → 인스타용/유튜브용 (자막은 | 로 줄 구분). 스타일은 날짜별 자동 로테이션
npm run edit -- --in 원본.mp4 --caption "6월27일(토요일)|주말 근무 후|인사드립니다"

# 특정 스타일 고정
npm run edit -- --in 원본.mp4 --caption "비가 와도|인사 드립니다" --style box
```

스타일·규격(위치·크기·색, 명함)은 `config/video-style.json`, 자막 디자인은 `templates/caption.html` 에서 조정합니다.

### 준비물
- **ffmpeg** (영상 합성)
- Chromium + 한글 폰트 (자막 렌더 — 썸네일과 동일)

---

## ⚠️ 원본 영상 전달 관련

채팅에 연결된 구글 드라이브 도구는 **한 파일당 10MB**까지만 받을 수 있어, 21~143MB인 원본 영상은
이 대화창에서 직접 처리하지 못합니다. **실제 운영(배포된 자동화)**에서는 정식 구글 드라이브 API(용량 제한 없음)로
영상을 받아 처리합니다. 이 폴더의 스크립트는 영상 파일 경로만 주면 동일하게 동작합니다.
